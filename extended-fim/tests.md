# Tests for Extended FIM (Wazuh)

Этот документ содержит тестовые сценарии для проверки работы расширенного File Integrity Monitoring (FIM) на Windows-агенте Wazuh.

Цель тестов — убедиться, что выбранные директории и типы файлов корректно фиксируются системой и отображаются в Wazuh Dashboard.

---

# Сценарий 1 — Создание .exe в AppData\Local

**Цель:** проверить фиксацию появления исполняемого файла в пользовательской директории.

1. Перейти в:  
   `C:\Users\<user>\AppData\Local\`
2. Создать файл `test_exec.exe`.
3. Проверить, что событие отображается в Wazuh.

**Ожидаемый результат:**  
Wazuh фиксирует создание файла как FIM event.

**Итог:** results/scenario1_create_exe.md

---

# Сценарий 2 — Создание скрипта в Temp

**Цель:** убедиться, что система фиксирует появление скриптовых файлов.

1. Перейти в:  
   `C:\Users\<user>\AppData\Local\Temp\`
2. Создать файл `test_script.ps1` или `test_script.bat`.
3. Проверить событие в Wazuh.

**Ожидаемый результат:**  
Появление нового .ps1 / .bat отражается в Dashboard.

**Итог:** results/scenario2_create_script.md

---

# Сценарий 3 — Создание .dll в ProgramData

**Цель:** проверить фиксацию появления библиотечных файлов.

1. Перейти в:  
   `C:\ProgramData\`
2. Создать файл `test_lib.dll`.
3. Проверить событие в Wazuh.

**Ожидаемый результат:**  
Wazuh фиксирует создание файла .dll.

**Итог:** results/scenario3_create_dll.md

---

# Сценарий 4 — Удаление любого тестового файла

**Цель:** проверить, как Wazuh отображает удаление ранее созданных файлов.

1. Удалить любой файл из сценариев 1–3.
2. Проверить событие удаления в Wazuh.

**Ожидаемый результат:**  
Должно появиться FIM-событие с типом "deleted".

**Итог:** results/scenario4_delete_file.md

Готово к использованию.
