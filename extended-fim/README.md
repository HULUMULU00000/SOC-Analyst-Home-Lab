# Extended FIM — Расширенный мониторинг целостности файлов (Wazuh)

Этот проект является развитием основной SOC-лаборатории и посвящён расширению File Integrity Monitoring (FIM) на Windows-агенте Wazuh.  
Цель — повысить глубину мониторинга и научиться отслеживать изменения, связанные с потенциально вредоносной активностью.

---

## Цели проекта

- Добавить мониторинг чувствительных директорий Windows (AppData, ProgramData, Temp).
- Включить отслеживание появления и изменения исполняемых и скриптовых файлов (.exe, .dll, .ps1, .bat, .vbs).
- Проверить корректность работы FIM после расширения.
- Подготовить реалистичные сценарии тестирования и выполнить анализ событий.

Проект позволяет понять, как злоумышленники взаимодействуют с файловой системой Windows, и как такие действия могут быть обнаружены на уровне SIEM.

---

## Структура проекта

extended-fim/
- README.md ← описание проекта  
- config_windows_fim.md ← план конфигурации FIM (директории, типы файлов)  
- fim_config_draft.xml ← черновик новой конфигурации для агента 
- test_scenarios.md ← сценарии тестирования расширенного FIM 
- analysis_template.md ← шаблон итогового отчёта 
- results/ ← фактические результаты тестов 

---

## Краткое описание процесса

1. Определяются чувствительные директории и расширения файлов, требующие мониторинга.  
2. Формируется черновик конфигурации и импортируется в `ossec.conf` Windows-агента.  
3. Агент перезапускается, и начинается сбор новых событий FIM.  
4. Проводятся тесты на создание, изменение и удаление файлов в выбранных директориях.  
5. Результаты фиксируются в папке `results/`.  
6. На основе собранных событий формируется итоговый отчёт с выводами.
